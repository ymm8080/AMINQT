# Skill: superpowers (编码前检查清单)

> **来源**: EWM Robot 项目 `Superpowers.md`
> **用途**: 开始任何重要工作前，逐维检查，防止架构/安全/性能/数据流问题
> **触发**: 新功能开发、重大重构、架构设计、疑难排查前

## 编码前检查清单

### 架构
- [ ] 符合现有架构模式？（分层：L1交互→L2策略→L3数据→L4执行）
- [ ] 是否引入新依赖？是否必要？
- [ ] 能扩展吗？（股票池从 5 只扩展到全市场）
- [ ] 有更简单的方案吗？
- [ ] 失败模式是什么？

### 测试
- [ ] 受影响代码有测试吗？
- [ ] 需要新增/更新测试吗？
- [ ] 边界情况测了？（空 DataFrame、单行数据、NaN 值、除零）
- [ ] 能单元测试隔离吗？
- [ ] 需要集成测试吗？

### 安全（量化特有）
- [ ] 未来函数防护？（铁律一，见 `rules/010`）
- [ ] 风控过滤是否在下单前执行？（铁律二）
- [ ] T+1 限制是否强制？（铁律四）
- [ ] 凭据是否从 `.env` 读取？（铁律五）
- [ ] 审计日志是否 append-only？（铁律六）

### 性能
- [ ] 有 N+1 读取吗？（逐个文件读 CSV = 禁止，见 ADR-003）
- [ ] 需要缓存吗？（内存缓存 `Dict[str, pd.DataFrame]`）
- [ ] 选股响应 < 5 秒？
- [ ] 内存占用合理吗？（全市场约 100MB）
- [ ] 有可以并行的操作吗？

### 可维护性
- [ ] 代码自解释吗？（清晰命名、简单逻辑）
- [ ] 复杂部分有注释/docstring 吗？
- [ ] 错误处理一致且有帮助吗？
- [ ] 6 个月后还能理解吗？
- [ ] 有重复代码应提取吗？

### 数据流
- [ ] 数据从哪来？到哪去？（akshare → CSV → 内存缓存 → 模型 → 选股）
- [ ] 边界处有验证吗？（列名映射在适配器内完成）
- [ ] 数据缺失/无效时怎么办？
- [ ] 有并发/竞态问题吗？
- [ ] 数据转换在所有情况下正确吗？

### 部署
- [ ] 需要数据库迁移吗？（此项目无 DB）
- [ ] 环境变量配置了吗？（`.env` 文件）
- [ ] 有回滚计划吗？（模型权重备份）
- [ ] 破坏向后兼容吗？

### 可观测性
- [ ] 日志恰当吗？（不多不少，关键步骤 INFO，异常 ERROR）
- [ ] 错误有上下文吗？（股票代码、时间戳、错误原因）
- [ ] 需要指标/告警吗？（回撤告警、下载失败告警）
- [ ] 能监控此功能健康状态吗？

## 红旗（停止并审查）

- 🚩 无测试
- 🚩 函数 > 50 行
- 🚩 嵌套条件 > 3 层
- 🚩 无错误处理
- 🚩 魔法数字/字符串（硬编码阈值，应放 `config/settings.py`）
- 🚩 复制粘贴代码
- 🚩 无日志
- 🚩 紧耦合
- 🚩 纯函数有副作用
- 🚩 修改共享状态

## 快速规则

✅ 编码前先思考 — 5 分钟规划省 5 小时调试
✅ 挑战假设 — "不可能失败" 是遗言
✅ 考虑用户 — 终端用户和开发者用户
✅ 留下更好的代码 — 顺手修小问题
✅ 知道何时停止 — 完美是好的敌人
