# Skill: diagnose (系统化调试)

> **来源**: EWM Robot 项目 `diagnose.md`
> **用途**: 系统化调试方法论，防止乱猜乱改
> **触发**: 任何 bug 排查、异常诊断、性能问题定位

## 系统化调试流程

### 步骤 1：复现问题
- 获取可复现的测试用例
- 记录触发问题的精确步骤
- 找到最小复现路径
- 确认稳定复现

### 步骤 2：收集信息
- 检查错误信息和堆栈
- 查看最近改动 (`git diff`)
- 检查日志
- 问：什么变了？什么时候开始的？

### 步骤 3：隔离根因
- 形成假设
- 测试假设
- 错了就换新假设
- 二分法排查（注释掉一半代码，还报错吗？）
- 加策略性日志断点

### 步骤 4：修复
- 修**根因**，不是症状
- 做最小改动
- 写一个能捕获此 bug 的测试
- 考虑边界情况

### 步骤 5：验证
- 运行复现用例 — 应通过
- 运行全部测试 — 不应破坏
- 测试相关功能
- 监控副作用

## 调试原则

✅ **理解再改** — 不要猜，要知道为什么坏了
✅ **一次一个改动** — 多个改动掩盖真正的修复
✅ **记录过程** — 写下试过什么
✅ **橡皮鸭** — 对着解释问题

## 量化场景常见调试

### 模型推理返回 NaN
```
1. 复现: predict(600519) → NaN
2. 收集: 检查输入 X 是否有 NaN
3. 隔离: np.nan_to_num(X) 是否执行了？
4. 修复: 在 build_features() 末尾加 np.nan_tonum(X)
5. 验证: predict(600519) → 0.85 ✓
```

### 选股超时 (>5s)
```
1. 复现: POST /select → 超时
2. 收集: 检查 DATA_CACHE 是否为空
3. 隔离: 是否逐文件读取 CSV？（违反 ADR-003）
4. 修复: 确认 startup 时全量加载到内存
5. 验证: POST /select → 2.3s ✓
```

### 未来函数泄漏
```
1. 复现: 回测收益虚高
2. 收集: 检查 build_features 中是否有 shift(-k)
3. 隔离: 验证第 t 天指标是否依赖 t+k 天数据
4. 修复: 改为 rolling/expanding
5. 验证: 修改第 t 天数据，第 t-1 天指标不变 ✓
```

## 反模式

❌ **散弹枪调试** — 随机改东西碰运气
❌ **治症不治本** — 止痛不治病
❌ **假设** — "不可能是 X" → 往往就是 X
❌ **忽略测试** — 测试通过但有 bug → 测试写错了

## 卡住时

1. 休息一下（换个视角）
2. 找人橡皮鸭
3. 检查是否已知问题
4. 查文档
5. 简化问题（最小复现）
6. 睡一觉
