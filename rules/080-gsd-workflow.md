# 080 — GSD 快速交付工作流 (Get Shit Done)

> **来源**: EWM Robot 项目 `gsd-workflow.mdc` (alwaysApply: true)
> **适配**: 量化交易分阶段开发
> **核心**: Done > Perfect。先交付，再改进。

## 核心理念

**Done > Perfect。先交付，再改进。**

## 执行流程

### 1. 明确目标（2 分钟）
- 最小可行产出是什么？
- "完成"长什么样？
  - Phase 1 完成 = `data/raw/` 出现 5 个 CSV
  - Phase 2 完成 = X.shape[2] ≥ 25 且无 NaN
  - Phase 3 完成 = Loss < 0.01 且权重文件生成

### 2. 最小化计划（5 分钟）
- 关键步骤 3-5 步？
- 最快路径是什么？
- 什么可以跳过？

### 3. 执行力
- 时间盒：25 分钟冲刺
- 一次只做一件事
- 够好 > 完美：先交付能跑的东西

### 4. 回顾迭代
- 能用吗？✓ 安全吗？✓ 以后还能改进吗？✓
- 交付

## 决策框架

卡住时问自己：

1. **能轻松撤销吗？**
   - 能 → 30 秒决策，继续
   - 不能 → 最多 5 分钟

2. **出错的代价？**
   - 小（<1 小时修复）→ 直接干
   - 中（<1 天修复）→ 想 2 分钟
   - 大（大重构/实盘交易逻辑）→ 想 5 分钟

3. **6 个月后还重要吗？**
   - 重要 → 值得时间投入
   - 不重要 → 快速搞定

## 代码演进

1. 让它能跑（丑、硬编码、无所谓）
2. 让它正确（测试、重构、整洁）
3. 让它快（优化、缓存、索引）

**绝不跳过第一步去够第三步。**

> [量化] Phase 1-2 先用 5 只股票跑通全流程 → Phase 4 扩展到全市场。不要一开始就全市场遍历。

## Phase 对应关系

| GSD 步骤 | AMINQT Phase | 完成 = |
|----------|-------------|--------|
| 能跑 | Phase 1 数据下载 | 5 个 CSV 存在 |
| 能跑 | Phase 2 因子工程 | X.shape 正确 |
| 正确 | Phase 3 模型训练 | Loss < 0.01 |
| 正确 | Phase 4 风控+API | select < 5s |
| 快 | Phase 5 可视化 | 面板可交互 |
| 正确 | Phase 6 实盘 | 模拟执行器通过 |

## 时间配比
- 60% 构建
- 20% 测试
- 10% 清理
- 10% 文档
