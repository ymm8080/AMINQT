# 000 — 全局铁律 (Global Iron Rules)

> **适用范围**: 所有代码、配置、文档、AI 生成内容
> **优先级**: 最高 — 与其他规则冲突时，以铁律为准

## 铁律一：未来函数绝对禁止

计算第 t 天的指标时，**严禁使用 t 以后的数据**。

- ✅ 使用 `pandas.rolling(window=N).mean()` — 只看过去 N 天
- ✅ 使用 `pandas.expanding().mean()` — 累积到当前
- ❌ 严禁 `df.shift(-k)` — 引入未来数据
- ❌ 严禁用全量数据的 `fit_transform` 后再切片（信息泄漏）

**违反后果**: 模型回测表现虚高，实盘必亏。这是量化交易最致命的 bug。

## 铁律二：实盘资金安全

AUTO 模式下单前，必须经过 `risk_filter.apply_filters()` 硬约束过滤。

| 约束 | 阈值 | 动作 |
|------|------|------|
| 成交额 | < 5000万 | 剔除 |
| 涨跌幅 | \|x\| > 9.5% | 剔除 |
| 账户回撤 | > 3% | 返回空列表（停止一切下单） |

- 账户回撤 > 3% → 系统进入"熔断"状态，停止所有新仓
- 熔断后需**人工恢复**，系统不得自动解除

## 铁律三：LLM 交易禁令

LLM **不得直接发出实盘交易指令**。LLM 只建议；用户或风控引擎执行。

- LLM 输出的是"建议"，不是"指令"
- 实盘下单必须经过 `executor` 层，由风控引擎把关
- MANUAL 模式：弹窗推荐，用户确认后执行
- AUTO 模式：风控通过后自动下单，但 LLM 不参与下单决策

## 铁律四：T+1 限制

当日买入的股票**不能在当日卖出**。`executor` 必须强制执行此规则。

- 执行器需维护 `purchase_date` 字段
- 卖出时检查 `today > purchase_date`
- 违反 T+1 的卖单必须被拒绝并记录审计日志

## 铁律五：数据主权

凭据只从环境变量 / `.env` 加载，严禁硬编码。

- iFinD 密码、券商账号、API Key 全部走 `os.getenv()`
- `.env` 文件不入 git (`.gitignore`)
- 配置文件中只有路径和常量，无敏感信息

## 铁律六：审计日志不可变

交易日志 **append-only**，严禁 DELETE 或 UPDATE。

- 每笔交易记录：时间戳、代码、方向、数量、价格、模式
- 交易日志用于事后审计和回测验证
- 修改日志 = 篡改交易记录，绝对禁止

## 铁律七：变更前备份

修改模型权重、配置或数据库 schema 前，必须先备份。

- 模型权重：复制 `.pth`/`.pkl` 到 `trained/backup/`
- 配置文件：git 版本控制 + 手动备份
- 数据文件：备份到 `data/backup/`
