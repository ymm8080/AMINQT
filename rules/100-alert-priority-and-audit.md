# 100 — 告警分级与审计合规 (Alert Priority + Audit Compliance)

> **来源**: EWM Robot 项目 `080-enterprise-policies.mdc` (alwaysApply: true)
> **适配**: 删除了不适用的飞书/短信/2FA 企业基础设施，保留告警分级和审计日志保留期

## 1. 告警优先级矩阵

| 级别 | 触发条件 | 响应时间 | 输出方式 |
|------|----------|----------|----------|
| **P0** (资金安全) | 账户回撤 > 3%、实盘下单异常 | 立即 | 终端 ERROR + 日志 + 停止下单 |
| **P1** (功能降级) | 模型推理失败、数据下载失败、调度器异常 | 5 分钟内 | 终端 WARNING + 日志 |
| **P2** (日常事件) | 单只股票跳过、缓存刷新完成 | 10 分钟内 | 日志 INFO |

## 2. 告警冷却

- 同一告警原因 **30 分钟内最多 1 次**
- 冷却期内日志仍记录，但不重复输出到终端
- 超限后仅写日志，不刷屏

## 3. 审计日志合规

- 交易审计日志 **保留 180 天**
- 日志格式: JSON (append-only，见 `rules/030-risk-filter-hard-constraints.md`)
- 严禁 DELETE 或 UPDATE 已写入的审计记录
- 超过 180 天的日志自动归档到 `logs/archive/`

## 4. 凭据安全

- iFinD 密码、券商账号: 只从 `.env` 读取，不入代码
- `.env` 不入 git (`.gitignore`)
- 严禁在日志中输出凭据明文
- 配置文件中只有路径和常量

## 5. 交易安全合规

- AUTO 模式下单前必须经过 `risk_filter.apply_filters()` (铁律二)
- T+1 限制必须由 `executor` 强制执行 (铁律四)
- 实盘交易日志必须包含: 时间戳、代码、方向、数量、价格、模式、风控结果
- 模拟执行器 (`SimExecutor`) 的审计日志格式与实盘一致
